摘要

本文详细介绍了如何使用Facebook开发的fastText库进行中文短文本多分类任务。内容包括数据预处理、模型训练、参数调优和性能评估等完整流程。fastText以其训练速度快、内存占用少的特点，特别适合处理大规模文本分类任务。

1. 引言

在风控内容安全领域，文本分类是一项基础且重要的任务。传统的机器学习方法如SVM和随机森林需要复杂的特征工程，而深度学习模型如BERT虽然效果优秀但计算成本高。fastText作为Facebook开发的一个文本分类和词向量训练工具，在保证较高准确率的同时，具有训练速度快、资源消耗少的优势。

本文重点介绍如何利用fastText处理中文短文本多分类问题，包括数据预处理、模型训练、评估和优化等完整流程。我们以内容审核场景为例，将文本分为"正常"、"广告"、"辱骂"、"政治敏感"和"其他违规"五个类别。

适用场景: fastText特别适合处理大规模文本分类任务，当需要快速迭代或资源有限时，它是一个优秀的选择。

2. fastText简介

2.1 fastText特点

fastText是Facebook于2016年开源的一个文本处理库，具有以下特点：

  • 训练速度快: 比深度学习模型快几个数量级
  • 内存占用少: 适合在资源受限的环境中部署
  • 支持子词嵌入: 通过n-gram考虑词内部结构，处理未登录词效果好
  • 简单易用: 命令行和Python接口都很简单

2.2 工作原理

fastText使用词袋模型表示文本，并添加了n-gram特征以捕捉局部词序信息。其模型结构类似于浅层神经网络，通过层次Softmax加速训练过程。

文本输入 → 词向量平均 → 隐藏层 → 输出层

+ n-gram特征

注意: fastText默认使用分层Softmax，这在处理多分类问题时可以大幅减少计算量。

fastText要求训练数据每行一个样本，格式为：

文本格式

例如：

文本格式

__label__政治敏感 讨论国家领导人是不合适的行为

3.2 中文文本预处理

中文文本需要先进行分词处理。我们使用jieba分词器：

Python 
import jieba

def preprocess_chinese_text(text):
    # 分词
    words = jieba.cut(text)
    # 过滤停用词和标点符号
    filtered_words = [word for word in words if word.strip() and word not in stop_words]
    return " ".join(filtered_words)

# 示例
text = "今天的天气真不错，适合外出散步"
processed_text = preprocess_chinese_text(text)
print(processed_text)  # 输出: 今天 天气 不错 适合 外出 散步

3.3 数据划分

将数据划分为训练集、验证集和测试集：

Python 
import pandas as pd
from sklearn.model_selection import train_test_split

# 读取数据
data = pd.read_csv('labeled_texts.csv')

# 划分训练集和测试集
train_data, test_data = train_test_split(data, test_size=0.2, random_state=42)

# 划分训练集和验证集
train_data, val_data = train_test_split(train_data, test_size=0.1, random_state=42)

# 保存为fastText格式
def save_fasttext_format(data, filename):
    with open(filename, 'w', encoding='utf-8') as f:
        for _, row in data.iterrows():
            label = f"__label__{row['label']}"
            text = preprocess_chinese_text(row['text'])
            f.write(f"{label} {text}\n")

save_fasttext_format(train_data, 'train.txt')
save_fasttext_format(val_data, 'val.txt')
save_fasttext_format(test_data, 'test.txt')

4.3 参数调优

fastText提供多个可调参数，以下是一些重要参数：

Python 
# 使用调优参数训练模型
model = fasttext.train_supervised(
    input='train.txt',
    lr=0.5,            # 学习率
    epoch=25,          # 训练轮数
    wordNgrams=2,      # 使用2-gram特征
    dim=100,           # 词向量维度
    loss='softmax',    # 损失函数，可选'softmax'或'hs'（分层softmax）
    minCount=5,        # 词频阈值，低于此值的词会被忽略
    verbose=2          # 输出详细日志
)
参数说明:
  • wordNgrams: 使用n-gram特征，建议值2-3
  • dim: 词向量维度，通常100-300
  • loss: 小数据集用softmax，大数据集用hs

4.4 自动超参数优化

fastText支持自动超参数优化：

Python 
# 自动超参数优化
model = fasttext.train_supervised(
    input='train.txt',
    autotuneValidationFile='val.txt',
    autotuneDuration=300  # 调优时间（秒）
)

# 保存最佳模型
model.save_model('model_tuned.bin')

5. 性能评估

5.1 基础评估

使用测试集评估模型性能：

Python 
# 在测试集上评估
def evaluate_model(model, test_file):
    result = model.test(test_file)
    print(f"样本数: {result[0]}")
    print(f"准确率: {result[1]}")
    print(f"召回率: {result[2]}")

    # 获取每个类别的精确率和召回率
    with open(test_file, 'r', encoding='utf-8') as f:
        texts = f.readlines()

    labels = [line.split(' ')[0] for line in texts]
    predictions = [model.predict(line.split(' ')[1:])[0][0] for line in texts]

    from sklearn.metrics import classification_report
    print(classification_report(labels, predictions))

    return result

evaluate_model(model, 'test.txt')

6. 部署与应用

6.1 模型部署

使用训练好的模型进行预测：

Python 
# 加载模型
model = fasttext.load_model('model_tuned.bin')

# 单个文本预测
def predict_text(text):
    processed_text = preprocess_chinese_text(text)
    labels, probabilities = model.predict(processed_text, k=3)  # 返回top-3预测
    return labels, probabilities

# 示例
text = "全新iPhone特价优惠，只要2999元！"
labels, probs = predict_text(text)
print(f"预测标签: {labels}")
print(f"对应概率: {probs}")

6.2 批量处理

对于大规模数据，可以使用批量预测：

Python 
import pandas as pd
from tqdm import tqdm

def batch_predict(input_file, output_file):
    # 读取数据
    data = pd.read_csv(input_file)

    # 预测并保存结果
    results = []
    for text in tqdm(data['text']):
        labels, probs = predict_text(text)
        results.append({
            'text': text,
            'predicted_label': labels[0],
            'probability': probs[0],
            'top3_labels': labels,
            'top3_probabilities': probs
        })

    # 保存结果
    result_df = pd.DataFrame(results)
    result_df.to_csv(output_file, index=False, encoding='utf-8-sig')

    return result_df

batch_predict('unlabeled_data.csv', 'classification_results.csv')

6.3 API服务

使用Flask创建简单的分类API：

Python 
from flask import Flask, request, jsonify
import fasttext

app = Flask(__name__)
model = fasttext.load_model('model_tuned.bin')

@app.route('/classify', methods=['POST'])
def classify_text():
    data = request.get_json()
    text = data.get('text', '')

    if not text:
        return jsonify({'error': 'No text provided'}), 400

    # 预处理和预测
    processed_text = preprocess_chinese_text(text)
    labels, probabilities = model.predict(processed_text, k=3)

    # 格式化结果
    result = {
        'text': text,
        'predictions': [
            {'label': label.replace('__label__', ''), 'probability': float(prob)}
            for label, prob in zip(labels, probabilities)
        ]
    }

    return jsonify(result)

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=5000)

7. 总结

本文详细介绍了使用fastText进行中文短文本多分类的完整流程。fastText以其训练速度快、资源消耗少的优势，特别适合处理大规模文本分类任务和实时应用场景。

7.1 优势与局限

优势:

  • 训练和预测速度快
  • 内存占用少，适合资源受限环境
  • 对未登录词处理效果好（得益于n-gram特征）
  • 简单易用，上手快

局限:

  • 无法捕捉长距离依赖关系
  • 对词序信息利用有限
  • 在处理复杂语义任务时准确率低于深度学习模型

7.2 应用场景

fastText特别适合以下场景：

  • 大规模文本分类任务
  • 实时内容审核系统
  • 资源受限环境（如移动端）
  • 快速原型开发和迭代
未来工作: 未来我们将探索fastText与其他模型（如BERT）的集成方案，结合两者的优势，在保持高效的同时提升分类准确率。
